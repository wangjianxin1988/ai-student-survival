## 三、更新流程

### 步骤1: 检查政策变更
```bash
# 访问各大学官方页面
# 1. 搜索大学官网的AI policy页面
# 2. 检查是否有新的policy更新
# 3. 记录变更内容和日期
```

### 步骤2: 更新数据库
```sql
-- 在Supabase SQL编辑器中执行
-- 方式A: 更新现有记录
UPDATE university_policies
SET
  overall_policy = 'allowed',
  assignment_policy = '更新后的政策内容',
  last_updated = '2025-05-16',
  updated_at = NOW()
WHERE university_slug = '大学slug';

-- 方式B: 新增大学
INSERT INTO university_policies (...) VALUES (...);
```

### 步骤3: 验证更新
```bash
# 运行构建验证
npm run build

# 检查API响应
curl https://your-site.com/api/policies | jq '.meta.total'
```

## 四、政策分类标准

| 分类 | 定义 | 示例 |
|------|------|------|
| allowed | 明确允许使用AI | MIT, Stanford, HKU |
| restricted | 有条件限制使用 | Harvard, Oxford |
| prohibited | 明确禁止使用 | Oxford (部分场景) |
| case_by_case | 各课程自定 | UCL, Brown |

---

## 五、AI工具分类

### 允许工具 (Allowed Tools)
- ChatGPT (OpenAI)
- Claude (Anthropic)
- GitHub Copilot (Microsoft)
- Gemini (Google)
- Perplexity
- Midjourney / DALL-E (图像)
- Copilot (Microsoft)

### 限制工具 (Restricted Tools)
- 需要申请审批的工具
- 有特定使用场景限制的工具

### 禁止工具 (Prohibited Tools)
- 完全禁止使用的工具
- 可能导致学术不端的工具
